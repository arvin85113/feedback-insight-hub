# 系統架構說明 - Feedback Insight Hub
> 生成日期：2026-04-22

---

## 系統概覽

本系統為一套**問卷回饋洞察平台**，提供問卷建立、回饋收集、統計分析、文字分析、改善追蹤與通知管理等功能。系統分為**終端用戶**與**管理者**兩種角色，後端採用 Django 框架，並整合資料庫進行資料管理。

---

## 使用者角色

### 終端用戶
- 透過 QR Code 掃描連結進入問卷頁面
- 註冊後直接進入問卷填寫
- 可在客服中心檢視、追蹤問卷紀錄
- 接收更新通知
- 可檢視近期回覆紀錄，並可複製填寫過的問卷

### 管理者
- 透過後台權限進行所有問卷與分析操作
- 管理問卷、統計分析、文字分析、改善追蹤、通知中心

---

## 核心功能模組

### 1. 首頁
- 提供 QR Code 掃描入口，產生問卷網頁連結
- 導向問卷頁面供用戶填寫

---

### 2. 問卷管理

#### 建立問卷
- 創建題目，支援以下題型：
  - **填空題**
  - **選擇題**
  - **量化資料**（文字分析）
- 創建題目時可預先填寫資料類型，便於後續搭配統計方法自動匹配

#### 題目類型分類
| 類型 | 子類型 |
|------|--------|
| 計量資料 | 區間資料、比例資料 |
| 非計量資料 | 名目資料、順序資料 |

- 計量/非計量資料可進行**半自動分類**

#### 問卷列表
- 編輯問卷
- 描述統計

---

### 3. 統計分析

快速檢視每個題目的資料類型與分佈，分析建議與基礎分布，作為決策前的第一層閱讀面板。

#### 分析流程
1. 選擇要分析的問卷
2. 選擇一個題目（主題目）
3. 自動列出「可搭配分析的題目」
4. 顯示每個配對對應的分析方法
5. 使用者選擇欲進行真題的分析
6. 系統檢查資料條件（通過 / 不通過）
7. **通過** → 分析執行 → 顯示結果與解釋
8. **不通過** → 顯示錯誤訊息

#### 支援統計方法
- 卡方檢定
- 獨立樣本 t 檢定
- 配對樣本 t 檢定
- 單因子 ANOVA
- Pearson 相關
- Spearman 相關

---

### 4. 文字分析

整理開放式題目的關鍵字與主題分類，協助團隊快速理解客戶回饋的重點方向。

#### 分析流程
1. 選擇問卷
2. 接收量化資料
3. 輸出：
   - **文字雲**：方便瀏覽常見回饋
   - **詞頻**：可自訂出現次數閾值，超過 N 次後自動建立類別
4. 支援食宿或預設關鍵詞（UI、口味、服務態度等）

---

### 5. 改善追蹤

集中管理所有改善項目，快速查看其對應問卷、通知狀態與最近更新節點。

#### 功能項目（側邊欄）
- 問卷總數
- 有效回覆
- 改善項目
- 平均回覆量
- 回覆量趨勢
- 營運洞察
- 重點問卷
- 近期改善任務
- 最新回覆

#### 改善流程
1. 選擇問卷
2. 手動新增改善項目
3. 產生改善列表

---

### 6. 通知中心

查看哪些改善更新已對外發布，並用同一個列表掌握通知素材與時間點。

#### 通知流程
1. 改善通知
2. 追蹤有提出此改善的使用者
3. 針對提出相關回饋的使用者發送特別感謝信
4. 針對開啟通知功能的使用者推送更新消息 → **通知**

---

## 後端技術

| 項目 | 技術 |
|------|------|
| 後端框架 | Django |
| 資料庫 | 關聯式資料庫（Django ORM） |
| 通知機制 | 站內通知 + Email |
| 問卷入口 | QR Code 連結 |

---

## 資料流向摘要

```
用戶掃描 QR Code
    → 進入問卷頁面填寫
    → 資料存入資料庫
    → 管理者於後台進行統計分析 / 文字分析
    → 產出改善項目
    → 通知中心發送更新通知給相關用戶
```

---

## 備註

- 本架構圖使用 Visual Paradigm 繪製
- 統計方法依資料類型自動匹配，並在執行前進行資料條件檢查
- 文字分析支援自訂關鍵詞與閾值設定
